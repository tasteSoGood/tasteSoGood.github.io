记2017年秋季PAT考试

2017-09-17

这次参加的是PAT的甲级考试，考题一共四道，分值分别是20，25， 25， 30，在平时刷题的过程中，并没有能在用时那么短的时间内写完四道题，所以这次考试的信心并不足。

考试因为系统的问题延迟了一个小时，过程中还是慢慢适应了考试环境。题目并没有我想象中的用到太多的算法，主要考察的能力还是逻辑和解决问题的方式。第一题因为延迟更换了一道，换题之前的那道题在正式开考时都差不多想明白了。不过题目换了，想也白想~

前三道题可以说都没有考到考纲中要求的算法。

第一题Cut Integer (20) ，要求把一个正整数 ZZZ 按位数分解为 AAA 和 BBB ，然后判断 ZA×B\dfrac{Z}{A\times B}A×BZ​ 是不是一个整数。这个问题的难点是分解的时候，可能下面的乘积会很大，超过long long的范围，所以需要在乘之前做一次约分（欧几里得算法），注意，在约分和分解的过程中，可能会出现 000 ，这时候应该直接判断后跳出，否则会发生浮点错误。

第二题Splitting A Linked List (25) ，一道静态链表的题目。在处理静态链表的时候，完全不用考虑真正链表的那些问题，所以本质上就是一个数组处理的问题。

在最开始，我尝试用和快排中 partition() 函数类似的方法做，但注意到，这道题中所要求的划分，是和原数组元素位置相关的，而 partition() 函数会使原数组顺序混乱。所以后来尝试用最原始的两次遍历加判断，将数组重排到新开辟的一个数组中，竟然直接通过了。

第三题Vertex Cover (25) ，这道题貌似是一道图论的题目，但实际编写的时候并没有用到任何图论算法。在考完后，听别人说用到 DFS ，我没细想，说不定那么写也是可以的。

题目给出一张无权无向图，然后再给出若干个点集合，要求这些点集合是否是原图的一个点覆盖，这里点覆盖的意思就是，如果说有一个点集，能够让图中的每一条边都触碰到这个点集，那么它就是一个点覆盖。

我的思路很简单，暴力求解，遍历每一条边，判断它们是不是都能连接到点集，如果可以就返回是，不行就返回否。但这个思路的复杂度很显然是O(n3)O(n^3)O(n3)，按照一贯的套路，这样的算法是会超时的，那么可以将点集建立为一个最简单的哈希表，把查找的复杂度降低，算法复杂度就是O(n2)O(n^2)O(n2) ，这样过得也很轻松。

最后一道题Is It A Red-Black Tree (30) ，红黑树的，看到就基本放弃了。其实如果明白红黑树的原理，判断一棵红黑树并不困难，还是怪自己基础不够扎实。

具体的代码实现我会在之后几天给出，顺便说一句，PAT大神真的太厉害，我第一题还没看完，就有人满分交卷是什么情况？！
