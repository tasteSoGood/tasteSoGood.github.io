正则表达式初步

2019-10-28

今天来学习一下正则表达式。

正则表达式是用来匹配特定模式的字符串用的。下面分几块来介绍正则表达式的匹配规则

对单个字符的匹配

单个字符的匹配可以并起来写，比如\d\d\d可以匹配连续三个数码，下面介绍对单个字符的模式匹配

普通字符

RegEx Pattern Example
a 字符a a
1 数码1 1
- 字符- -

特殊字符

RegEx Pattern Example
. 任意字符 !, a, 1, ...
\d 一个任意数码 1, 5, ...
\D 一个任意非数码 a, -, A, ...
\w 匹配一个字母或者数字或者下划线，等价于[_a-zA-Z0-9] a, A, _, 0
\W 匹配一个非字母或者数字或者下划线，等价于[^_a-zA-Z0-9] ?, ~, +, )
\s 匹配一个空白符，这里可以匹配空格，制表符，换行符，回车符
\S 匹配一个非空白符，相当于[^\s]

这里的\w在英文，特别是编程语言中可以理解为单词字符

转义字符

在正则表达式中有特殊含义的字符是不可以直接匹配的，需要通过\来转义，比如

RegEx Pattern
\\ \
\| |
\* *
\? ?

模式都是差不多的，这里就不一一列举了

匹配Unicode编码字符

这里就一定需要查表了。前面一直在讲的，都是关于英文的匹配，但我们所使用的汉字是在unicode编码中的，这就需要相应的匹配操作了。

比如说，汉字“哈”的unicode编码是54c8，那么查找它的方法就是\u54C8或者\u54c8

可以看到，这是不区分大小写的，其中，\x..或\X..匹配的是最大字符数为2的字符，而\u是4个，\U是8个

目前，大部分正则引擎都直接支持中文，除非是极为特殊的字符才需要用到这种方法来匹配

量词

在匹配了单个字符以后，可能会想要同时匹配多个，比如匹配连续三个数码，就要用\d\d\d，遇到想匹配三个或者四个数码的字符串情况，这样的方法就无能无力了，这就需要用到量词来限定单个字符匹配的数量

RegEx Pattern Example Pattern
{n} 连续匹配一个字符n次 a{3} aaa
? 匹配一个字符0次或一次 a? a, NULL
+ 连续匹配一个字符大于等于1次 a+ a, aaa
* 连续匹配一个字符大于等于0次 a* NULL, a, aaaaaaaaaaaa
{m, n} 连续匹配一个字符m次到n次 a{3, 5} aaaa, aaa, aaaaa
{m, } 连续匹配一个字符大于等于m次 a{3, } aaaa, aaa, aaaaaaaaaaa

正则表达式中的量词是具有贪心特性的，所谓“贪心”，是指量词会匹配所有能匹配的字符。所以说，像下面的这几种表达式，就会匹配出很多结果

惰性匹配和占有匹配

这里只说几点，懒惰匹配和占有匹配，因为大部分的我们之前都接触到了

RegEx Pattern
{n} 精确匹配n次
{n， } 精确匹配n次及以上
{m，n} 精确匹配m到n次
{0，1} 0次或1次
{1，0} 1次或更多
{0， } 0次或更多
?? 懒惰匹配0次或1次
+? 懒惰匹配1次或多次
*? 懒惰匹配0次或多次
{n}? 懒惰匹配n次
{n， }? 懒惰匹配n次及以上
{m，n}? 懒惰匹配m到n次

懒惰匹配的意思就是匹配值全部取小，越懒越好。

占有匹配的语法格式就是把懒惰匹配后面的?改成+，它的特点是不会像普通量词那样进行回溯，可以提高查找的效率。但相应的语法需要做一点改变，这里就不深入探讨了。

字符组

如果要匹配单个字符，用单个字符的匹配就可以了。但是如果要单次匹配某几个字符，上面的两节都无法表述，就要用到字符组，一些典型的字符组写法和意义如下

RegEx Pattern Example
[abc] 匹配字母a或b或c a, b, c
[013] 匹配数码0或1或3 0, 1, 3
[a-z] 匹配一个小写字母 a, j, z
[A-Z] 匹配一个大写字母 A, J, Z
[0-9] 匹配一个数码 0, 5, 9
[^abc] 匹配一个除了a，b，c之外的所有字符，^相当于非运算 ?, l, 5
[^\d] 匹配一个非数码字符 ?, l, 5
[a-zA-Z] 匹配一个字母 a, A, j, K

分组

捕获式分组

在某些匹配情况下，会需要对某个匹配到的字符重写，比如说匹配aabb这种模式的字符串，jjkk，bbaa这些都是需要匹配到的，那么就可以用捕获分组来实现。它可以对捕获到的字符串在某处重现

RegEx Pattern Example
(a)-\1\1 (...)内的内容即是捕获组，\1即对第一个捕获组进行匹配 a-aa
(\d)\1(\d)\2 匹配aabb型的数码串 1122, 0011

非捕获式分组

有时虽然我们可以不需要捕获的分组重现，但是对于有些模式来说，用分组的方式分成几个子模式来匹配也是一种常用的方法

在一般的情况下，捕获而不重现并不会对查询带来多少影响。但是面对巨大的文本量，全部按照捕获式来分组就会给内存带来很大的负担，因为正则引擎还会将分组存储起来等待之后被唤醒

当明确地知道某个分组不需要重现，那么可以使用非捕获式分组

RegEx Pattern
(?:<pattern>) 以非捕获式的方式将<pattern>模式分组

特殊文本单位

上面所表达的模式都是将文本作为一个连续字符串来看待的，相当于在一个纸带上顺序寻找。但是在文本中有很多文本单位，比如单词、段落、页等等，某些模式在这些文本对象的帮助下描述起来会更加方便

单词

RegEx Pattern
\b...\b 匹配一个单词，\b表示单词的边界，不占位

比如\b.{5}\b表示任意长度为5的字符串，\b\w{5}\b表示任意长度为5的单词

相对应的，\B...\B表示非单词边界

行

RegEx Pattern
^ 表示行的起始，不占位
$ 表示行的末尾，不占位

比如^$表示空行，^In.*表示以In开头的一行，are\.$表示以are.结束的行

或运算

或运算使得正则表达式可以并列地匹配多种模式

RegEx Pattern
<pattern-1>|<pattern-2> 匹配模式<pattern-1>或者<pattern-2>

这里的<pattern-n>可以是单个字符，字符组，单词，捕获组等等

但是或运算不可以写在字符组中，比如[a|b|c]，这个表示的是匹配a或者|或者b或者c

  • The|They|abc

    表示匹配The或者They或者abc

  • \b(t|T)(h|H)(e|E|ey|eir)\b

    表示匹配单词the,The,THE,they,their等等，这说明或运算也可以被捕获
