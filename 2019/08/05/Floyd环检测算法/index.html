  1. 判定算法

这就是Floyd环检测算法。直白地说，这个算法就像两个人在赛道上跑步，从出发的时候开始，A就比B跑得快，那么如果在之后的某个时间，A还能遇到B，那这个赛道肯定是有个圈的。

首先我们可以清楚地知道，上述方法的时间复杂度为O(m+n)O(m + n)O(m+n)，空间复杂度是O(1)O(1)O(1)，对于空间复杂度要求较高的情况，可以利用此方法来优化，用时间换空间。

进一步的问题是：以上的算法为何可行呢？

可行性分析

对环路问题抽象化可以这样来做：设有一个有限集合X={xi},i=1,⋯ ,nX = \{x_i\}, i = 1, \cdots, nX={xi​},i=1,⋯,n，有一个状态转移函数f:X→Xf: X\to Xf:X→X，它接受一个XXX中的对象，返回一个XXX中的对象。我们以XXX中的某个元素xix_ixi​为初始，不断地经过fff函数迭代

  1. 第一步：xi=y0x_i = y_0xi​=y0​
  2. 第二步：f(xi)=y1f(x_i) = y_1f(xi​)=y1​
  3. 第三步：f(f(xi))=f(2)(xi)=y2f(f(x_i)) = f^{(2)}(x_i) = y_2f(f(xi​))=f(2)(xi​)=y2​

如此进行下去，如果在第kkk步和第lll步（k≠lk\not=lk​=l）有yk=yly_k = y_lyk​=yl​，那么就称这个集合在状态转移函数fff下存在环路

拓展

Floyd环检测算法是一种通用的环路检测算法，只要将问题建模为状态集和状态转移函数这两个要件，就可以利用该算法求解

举个例子

happy number是这样的数：对于一个正整数，将它的每一位求平方再加起来，再将得到的结果中的每一位求平方再加起来，以此类推，这个过程将会有两种结果。第一种是在若干步之后，结果变为1；第二种是在若干步之后，结果进入一个无限循环的状态。我们称可以进入前一种情况的正整数为happy number。试设计一个算法判断某个数是否为happy number

这个问题中就存在这状态和状态转移两个要件，可以通过Floyd环检测算法来解决
