我这里使用的是python3.6，在python语言中，关于==这个符号的定义是比较两者之间的内容是否相同；而is是内置的运算符，用于比较两者是否是同一个对象。

==运算符在类内部的实现是重载__eq__方法，既然是重载，那么这样一个方法的行为将是多样的，对于不同的类（对象），它将会表现出不同的特性。在今天这个例子中，==并没有体现出出乎意料的结果，所以不对它过多研究。

is在官方说明文档中的含义是：The operators is and is not test for object identity: x is y is true if and only if x and y are the same object. Object identity is determined using the id().

以上的含义是，is的用法的检验两个对象是否相同，而检验的方法是使用id()。

果然是这样的。python中没有指针，但是python的变量名其实都是引用。特别是在复制一个对象的时候，实际上只是复制了一个引用，对象的实体并没有改变。换句话说，这个变量的存储位置是不变的。这也是python的一个重要的特性。

更加困惑的事情来了：为什么x is y在第一个例子和第二个例子中的结果不同呢？

  2. 含空格的字符串与不含空格的字符串的存储方式不同。

字符串驻留机制

python语言的实现实际上是C语言，在实现某种特性的时候，除了考虑到使用上的方便，还要考虑到一个运算的效率。作为一种通用的程序设计语言，对效率的提高通常要考虑到方方面面的因素。

python在语言中内置了对字符串的优化机制——intern机制，翻译成中文是字符串驻留机制。这并不是python语言独有的，这种方式最早出现在函数式编程语言Lisp中。以下是维基百科中的介绍：

In computer science, string interning is a method of storing only one copy of each distinct string value, which must be immutable.Interning strings makes some string processing tasks more time- or space-efficient at the cost of requiring more time when the string is created or interned. The distinct values are stored in a string intern pool.

在计算机科学中，字符串驻留是一种对独有字串仅存储一个副本的方法，这个字串必须是不变的。驻留字符串将会在创建或者存储一个字串的时候花费更多的时间、但在某些字符串操作中将获得更好的时间或空间效率。这些独有值将存储在字串驻留池中。

在python2中，intern()方法是内置的处理函数，但在python3中，这个方法已经移到了sys包里面。
